⚠️ Design Observations (Interview Gold)
✅ Strengths:
Clean separation of concerns
Reusable pipeline
Template-driven (very scalable)
Handles missing data gracefully
⚠️ Weaknesses / Improvements:
Sync file read (readFileSync)
Blocks the event loop
Better: async version
Regex injection risk
If variable contains special regex chars → bug
No caching
Templates reloaded every time
String-only variables
Limits flexibility (no nested structures)
